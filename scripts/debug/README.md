# Local Lambda Debugging

Runs Lambda handlers directly in Node.js without Docker or SAM, enabling breakpoint debugging in VS Code and IntelliJ.

## Prerequisites

- [Node.js](https://nodejs.org/en/) version 22 (via [NVM](https://github.com/nvm-sh/nvm))
- `ts-node` — installed via `npm install`
- [AWS CLI](https://docs.aws.amazon.com/cli/latest/userguide/install-cliv2.html) with `audit-dev` SSO profile configured
- [jq](https://jqlang.org/) — required by `fetch-env-vars.sh`
- Access to the dev AWS environment

## Setup

Run once after cloning:

```bash
bash scripts/debug/setup/install-run-configs.sh
```

This generates `.vscode/launch.json` with a debug configuration for each Lambda. It also prints the manual steps required to set up IntelliJ run configurations.

### Environment variables

`env-vars.json` is the local substitute for Lambda environment variables. Fetch the current values from the deployed dev stack:

```bash
aws sso login --profile audit-dev
bash scripts/debug/setup/fetch-env-vars.sh
```

Re-run this whenever the deployed configuration changes (e.g. new KMS key ARNs or bucket names). The script merges variables from all Lambda functions and applies local overrides (`BACKUP_ENCRYPTION_ENABLED=false`, etc.).

## VS Code

Open the Run & Debug panel (`⇧⌘D`), select a configuration (e.g. `Debug s3CopyAndEncrypt`), and press F5.

## IntelliJ

Follow the instructions printed by `install-run-configs.sh` to create a Node.js run configuration, then use the debug button (⌃D).

## Structure

- `invoke-local.ts` — entry point; loads `env-vars.json`, builds a mock Lambda context, and invokes the handler
- `events/<lambdaName>.json` — test event payload for each Lambda
- `env-vars.json` — environment variables injected at runtime (generated by `fetch-env-vars.sh`)
- `assets/` — static test assets (e.g. seeded S3 files)
- `setup/install-run-configs.sh` — generates IDE debug configurations
- `setup/fetch-env-vars.sh` — fetches env vars from the deployed dev Lambda functions

## Adding a new Lambda

1. Create a test event: `scripts/debug/events/<lambdaName>.json`
2. Add the deployed function name to the `FUNCTIONS` array in `setup/fetch-env-vars.sh`
3. Re-run setup to regenerate IDE configurations:

```bash
bash scripts/debug/setup/fetch-env-vars.sh
bash scripts/debug/setup/install-run-configs.sh
```

The install script auto-discovers lambdas from `src/lambdas/` — no other changes needed.

## Notes

- `BACKUP_ENCRYPTION_ENABLED` is set to `false` in `env-vars.json` — the backup KMS key is in a different AWS account and cannot be assumed locally
- Edit `events/<lambdaName>.json` to change the input event
- The S3 bucket name in `events/s3CopyAndEncrypt.json` must match either `AUDIT_TEMPORARY_BUCKET_NAME` or `AUDIT_BUCKET_NAME` in `env-vars.json`
